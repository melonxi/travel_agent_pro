问题阐述：
在使用本项目时，我发现了如下的一些问题。我的表达可能有时候觉得混乱，你应该从我的表达中总结出其核心观点。

1. 第一阶段时，主要目标是确定目的地。但在实际和旅行 Agent 进行交互的过程中，我发现当用户提出一些问题时，Agent 的回答内容常包含大量的无关内容，可能淹没用户的焦点。同时，Agent 给出意见之前并不能保证都有足够的外部信息的支撑。

2. 第三阶段时，存在两个问题。当前的流程把大交通和住宿放在最后确认。以我个人旅行的体验是，一般会确定旅行的时间，在旅行确定时间之后会确定大交通。当确定完大交通之后，就可以把注意力完全放在旅行内容的规划上。

3. 在写入旅行计划的框架时，当前的过程会试图一次性更新所有内容，可能会导致内容过长，应该采用按天多次更新的方式。

4. 旅行框架的搭建，需要有专门的提示词，因为每一天如何安排，是需要进行反复思考权衡的。

5. 在第五个阶段对每一天的旅行内容进行规划，其实更多的是一个路径规划问题，现在实际执行的提示词，我不知道有没有完全按照这个核心的目标进行设计。

在第三步旅行骨架搭建的过程当中，这一步在人类规划旅行的时候，是一个非常好非常耗时的过程。

它涉及到目的地的一些筛选。这个过程之前会锚定一些不变量，根据这些不变量，然后再综合各种用户分享，才能够给出最后的决断。

但是目前在进行规划的时候，我发现 agent 并没有实现一个非常科学的框架搭建的前置的信息收集安排的过程。

当前第一阶段存在越界行为，可能会因为 Agent 的引导导致用户输出所有信息。现在的第三阶段又存在不收敛的问题。
